## 1. 快速开始

### 方式一：直接运行

1. 下载对应平台的 `frp-panel` 二进制文件和 `config.yaml`
2. 编辑 `config.yaml`（详见下一节）
3. 运行：

```bash
./frp-panel -config config.yaml
```

### 方式二：Docker 运行

```bash
docker run -d \
  --name frp-panel \
  -p 8080:8080 \
  -v /path/to/config.yaml:/app/config.yaml \
  -v /path/to/data:/app/data \
  frp-panel:latest
```

## 2. 配置文件详解

配置文件 `config.yaml` 是启动前必须编辑的文件。以下是完整示例，**标注了必须修改的项目**：

```yaml
server:
  host: "0.0.0.0"       # 监听地址，一般不用改
  port: 8080             # 面板端口，按需修改
  mode: "debug"          # debug / release，生产环境用 release

database:
  driver: "sqlite"       # 目前仅支持 sqlite
  dsn: "frp-panel.db"   # 数据库文件路径

redis:
  addr: "localhost:6379"
  password: ""
  db: 0

jwt:
  # !! 必须修改：JWT 签名密钥，改为任意随机字符串 !!
  secret: "CHANGE-ME-TO-A-RANDOM-SECRET-KEY"
  expire_time: 24h       # 登录 token 有效期
  issuer: "frp-panel"

frp:
  default_version: "0.68.0"   # frp 版本
  github_mirror: "https://github.com"  # GitHub 下载镜像
  download_timeout: 300       # 下载超时（秒）
  # !! 必须修改：面板自身的 webhook 地址，改为实际部署地址 !!
  plugin_webhook_url: "http://YOUR-PANEL-IP:8080/api/plugin/webhook"
  # !! 必须修改：所有 frps 共用的认证 token，改为随机字符串 !!
  server_token: "CHANGE-ME-TO-A-RANDOM-TOKEN"

admin:
  email: "admin@example.com"  # 管理员邮箱，按需修改
  # !! 必须修改：管理员初始密码 !!
  password: "CHANGE-ME"
```

### 必须修改的 4 项

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `jwt.secret` | JWT 签名密钥 | `aB3x$kL9mN2pQ7rS` |
| `frp.plugin_webhook_url` | 面板 webhook 地址 | `https://panel.example.com/api/plugin/webhook` |
| `frp.server_token` | frps 认证 token | `myRandomToken123` |
| `admin.password` | 管理员密码 | `StrongP@ssw0rd` |

> **注意**：`plugin_webhook_url` 中的 IP 必须是面板服务器的公网 IP（或内网 IP，取决于 frps 能否访问到面板）。

## 3. 首次登录

1. 启动面板后，浏览器访问 `http://YOUR-IP:8080`
2. 使用配置文件中设置的管理员邮箱和密码登录
3. 登录后请立即修改密码（用户中心 → 修改密码）

## 4. 添加 frps 服务器

登录管理后台后，进入 **服务器管理** 页面，点击 **添加服务器**。

### 必填字段

| 字段 | 说明 |
|------|------|
| 名称 | 服务器标识，如 "美国节点" |
| IP 地址 | frps 服务器的公网 IP |
| 绑定端口 | frps 的 `bind_port`，默认 7000 |
| Dashboard 端口 | frps 的 `dashboard_port`，默认 7500 |
| Dashboard 用户名 | frps dashboard 的用户名，默认 `admin` |
| Dashboard 密码 | frps dashboard 的密码 |

### 可选字段

| 字段 | 说明 |
|------|------|
| Vhost HTTP 端口 | frps 的 `vhost_http_port`，用于 HTTP 代理 |
| Vhost HTTPS 端口 | frps 的 `vhost_https_port`，用于 HTTPS 代理 |
| Token | frps 的 `token`（如果 frps 配置了 token） |

### frps 端配置要求

frps 端需要启用 dashboard 并配置 httpPlugin 指向面板 webhook：

```toml
# frps.toml
bindPort = 7000

webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "your_dashboard_password"

# 指向面板的 webhook
httpPlugin.addr = "YOUR-PANEL-IP"
httpPlugin.port = 8080
httpPlugin.path = "/api/plugin/webhook"
httpPlugin.ops = ["Login", "NewProxy", "CloseProxy", "Ping"]
```

## 5. 用户注册与套餐

### 注册设置

在 **系统设置** 中可以控制是否开放注册：
- `registration_enabled`: 是否允许新用户注册
- `login_enabled`: 是否允许登录（维护时可关闭）

### 套餐管理

管理后台 → **套餐管理**，可以创建不同等级的套餐：

| 字段 | 说明 |
|------|------|
| 名称 | 套餐名，如 "基础版" |
| 最大代理数 | 用户可创建的代理数量上限 |
| 最大带宽 | 每个代理的带宽上限（字节/秒） |
| 最大流量 | 每月总流量上限（字节） |
| 最大端口数 | 可使用的端口数量 |
| 有效期（天） | 套餐有效天数 |
| 月付/季付/年付价格 | 0 表示免费 |

系统内置了 4 个默认套餐：Free、Basic、Pro、Enterprise。

## 6. 创建代理

用户登录后进入 **我的代理** 页面，点击 **创建代理**。

### 通用字段

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 代理标识 | 是 |
| 类型 | tcp / udp / http / https / stcp / xtcp | 是 |
| 本地 IP | 要转发的目标 IP，默认 127.0.0.1 | 否 |
| 本地端口 | 要转发的目标端口 | 是 |
| 启用加密 | 是否加密传输 | 否 |
| 启用压缩 | 是否压缩传输 | 否 |
| 带宽限制 | 代理级别的带宽限制（字节/秒） | 否 |

### 按类型填写

**TCP / UDP 代理**
- 远程端口（必填）：用户通过此端口访问你的服务

**HTTP / HTTPS 代理**
- 自定义域名：绑定的域名（如 `app.example.com`）
- 子域名：使用平台提供的子域名（如 `myapp.your-domain.com`）
- 两者至少填一个

**STCP / XTCP 代理**
- 密钥（必填）：访客端连接时需要的密钥

## 7. 连接方式

### 方式一：Frpc-Desktop 客户端（推荐）

1. 下载 Frpc-Desktop 客户端
2. 在面板 **个人中心** 复制你的 API Key
3. 在 Frpc-Desktop 中添加服务器，填入：
   - 服务器 IP
   - 绑定端口（默认 7000）
   - API Key（作为认证凭据）
4. 点击连接，客户端会自动拉取你的代理配置

### 方式二：官方 frpc 客户端

在代理详情页点击 **获取配置**，复制 frpc 配置文件内容，保存为 `frpc.toml` 后运行：

```bash
./frpc -c frpc.toml
```

## 8. 支持的支付方式

管理后台 → **支付配置**，目前支持：

| 支付方式 | 说明 |
|----------|------|
| 支付宝（当面付） | 需要签约当面付，填写应用 ID 和私钥 |
| USDT (TRC20) | 填写 TRC20 收款地址 |

### 支付宝配置

| 字段 | 说明 |
|------|------|
| App ID | 支付宝开放平台应用 ID |
| 私钥 | 应用 RSA2 私钥 |
| 支付宝公钥 | 支付宝提供的公钥 |
| 异步通知 URL | 面板回调地址，格式 `http://YOUR-IP:8080/api/payment/notify/alipay` |

### USDT 配置

| 字段 | 说明 |
|------|------|
| TRC20 地址 | 你的 USDT-TRC20 收款地址 |
| 汇率 | USDT 对人民币汇率（如 7.2） |

## 9. 邀请返利系统

用户在 **邀请好友** 页面获取邀请码，分享给好友注册后可获得返利。

### 设置

在 **系统设置** 中配置：
- `invite_rebate_level1_percent`: 一级返利比例（%），默认 10
- `invite_rebate_level2_percent`: 二级返利比例（%），默认 5

### 规则

- 好友通过邀请码注册后，其每次充值，邀请人都会获得返利
- 二级返利：被邀请人再邀请第三人，原邀请人也能获得二级返利
- 返利金额记录在 **返利记录** 中，管理员可手动处理提现

## 12. 监控与告警

### 服务器监控

面板每 30 秒自动轮询 frps dashboard API，获取：
- 代理连接状态
- 流量统计（上传/下载）
- 客户端连接数

### 告警规则

管理后台 → **告警管理**，可设置告警规则：
- 流量使用百分比告警
- 服务器离线告警
- 代理异常告警

### 流量限制

- 用户流量超限时，所有代理将自动禁用
- 套餐带宽上限在创建/修改代理时强制校验

## 13. 常见问题

### Q: frps 连接面板失败？

检查 `plugin_webhook_url` 是否正确，确保 frps 所在服务器能访问面板的 8080 端口。

### Q: 代理状态显示"未连接"？

1. 确认 frpc 客户端已启动并连接成功
2. 检查 frps dashboard 是否正常工作
3. 等待 30 秒（轮询间隔）后刷新

### Q: 如何修改管理员密码？

登录后进入 **个人中心** → **修改密码**。

### Q: 如何备份数据？

备份 `frp-panel.db` 文件即可（SQLite 数据库）。

### Q: 端口 1024 以下无法使用？

系统限制远程端口必须 >= 1024，这是安全策略，无法修改。

### Q: 如何升级 frp-panel？

1. 停止当前面板
2. 替换二进制文件
3. 重新启动（数据库会自动迁移）
